## 2026-07-09 - Add clear button to search inputs
**Learning:** Adding a clear button (X icon) to search inputs significantly improves usability by allowing users to quickly reset their query without repeatedly pressing backspace. It also provides immediate visual feedback that the field is active and can be cleared.
**Action:** Always include a clear button for search inputs, ensuring it is keyboard accessible, has an appropriate aria-label, and properly manages focus when clicked (returning focus to the input).
## 2026-07-10 - Add missing focus rings and ARIA labels to buttons\n**Learning:** Icon-only buttons used for secondary actions (like clearing search inputs or closing drawers) are frequently missing accessible focus states (`focus-visible:ring-2 focus-visible:ring-accent`) and ARIA labels. Users relying on keyboard navigation need clear visual indicators of their current focus.\n**Action:** Always test components with keyboard navigation (Tab key) to ensure all interactive elements, particularly icon-only buttons, have visible focus rings and appropriate `aria-label` attributes.
