## Overview
Terminal / Matrix（ターミナル）は、80年代のCrtモニター、古いUnixシステム端末、そしてマトリックスのようなコマンドラインコンソールのビジュアルを再現したスタイルです。
等幅フォントによるストイックな文字組み、完全な直角構造、そして点滅するカーソルアニメーションが、ハッカー風の知的なユーティリティ空間を創出します。
ライトモードでは琥珀色（アンバー）の古いペーパー風コンソールを、ダークモードでは漆黒の闇に鮮烈なグリーンが浮かぶモノクロマトリクスを表現します。

## Colors
システムの単一出力色（モノクローム）をベースにした、コントラストの明確な配色設計です。

### ライトモード (標準)
- **プライマリー/アクセント (`primary`):** 警告や強調を表すアンバー/琥珀色 (`#c67c00`)。
- **セカンダリー/テキスト/ボーダー (`secondary`):** 文字と罫線を描くディープアンバー (`#7c4f00`)。
- **ニュートラル/背景 (`neutral`):** クラシックな端末の文字盤や保護紙を思わせる琥珀ホワイト (`#fbf5e6`)。
- **カード背景 (`card`):** 背景と親和する薄いアンバーグレー (`#f2e9d3`)。

### ダークモード
- **プライマリー (ダーク):** 濃いフォレストグリーン (`#008800`)。
- **セカンダリー (ダーク):** 鮮烈なシステムグリーン (`#00ff00`)。すべてのテキストとボーダーがこの色で統一されます。
- **ニュートラル (ダーク):** 完全な漆黒 (`#000000`)。
- **カード背景 (ダーク):** ソリッドな極暗グレー (`#050505`)。

## Typography
すべての文字は均等な幅で整列する必要があります。
- **メインフォント:** `'Share Tech Mono'` を採用。デジタルで少しメカニカルな美しい等幅フォントが、コンソール画面のシステム的な味わいを引き立てます。

## Layout
等幅フォントのグリッド整列を活かし、テーブルやリストなどのテキストデータを整然とレイアウトします。
境界線は細い実線（`1px solid`）で構築され、すべてのコンポーネントが直線的に隙間なく並ぶような構造を作ります。
また、H2見出しの末尾には ` _` （点滅するアンダースコア）がアニメーションで付与され、常に入力を待機しているシェルのような質感をもたらします。

## Elevation & Depth
このテーマにおいて「影（ドロップシャドウ）」は不要な装飾として省略されます。
- **カードシャドウ:** なし (`none`)。
- **ホバー時挙動:** 影を落とす代わりに、カードにカーソルを合わせると `0 0 10px var(--color-border)` の緑（またはアンバー）の発光枠線（ボーダーグロー）が浮かび上がるハイテクな演出を行います。
- **ボタン挙動:** 通常時は枠線だけのフラットなボタンですが、ホバー時に背景をテキスト色（`secondary`）で完全に塗りつぶし、テキストを背景色（`bg`）に反転させるネガポジ表示を行います。クリック時には `inset 0 0 5px` の沈み込みシャドウを付与します。

## Shapes
すべてのデジタル端末がそうであるように、丸みは一切ありません。
- **角丸 (`rounded.card` / `rounded.btn`):** `0px` (完全な直角)。

## Components
### カード (`.theme-card`)
角丸のない直角カード。1pxのボーダーと、ホバー時の外枠ネオングロー効果。

### ボタン (`.theme-btn`)
角丸のない直角ボタン。等幅フォントが適用され、ホバー時のテキスト反転とクリック時のインナーシャドウが必須です。

## Do's and Don'ts
### Do's
- すべてのテキスト、見出し、ボタンには等幅フォント（`'Share Tech Mono'` または `ui-monospace`）を適用してください。
- H2などの見出しには、末尾で点滅するアンダースコア（`_`）のアニメーション効果を付与してターミナルの質感を再現してください。

### Don'ts
- 角丸（`border-radius`）は一切使用しないでください。
- 写真やグラデーションなどの有機的なビジュアル表現は、ハッカー風CLIコンソールの統一感を損なうため避けてください。
