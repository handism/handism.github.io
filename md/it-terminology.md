## IT用語集

| カテゴリ              | 用語                       | 説明                                                                                                           |
| --------------------- | -------------------------- | -------------------------------------------------------------------------------------------------------------- |
| インフラ/ネットワーク | `コロケーション`           | 同じ場所に複数の機器やサービスを配置すること。                                                                 |
| インフラ/ネットワーク | `パケットキャプチャー`     | ネットワーク上を流れるデータパケットを取得し解析すること。                                                     |
| インフラ/ネットワーク | `トランジット`             | データや通信が一つのネットワークを通過すること。                                                               |
| インフラ/ネットワーク | `ピアリング`               | 異なるネットワーク同士が直接接続されること。                                                                   |
| インフラ/ネットワーク | `バックボーンネットワーク` | 主要なネットワークの中心部分で、他のネットワークを結ぶ高容量のコアネットワーク。                               |
| インフラ/ネットワーク | `トポロジー`               | ネットワークやシステムの構造や配置に関する形状やパターン。                                                     |
| インフラ/ネットワーク | `流量制限`                 | ネットワーク上のデータの送受信速度を制限すること。                                                             |
| セキュリティ/認証     | `ウォーターマーク`         | 画像や文書に埋め込まれた識別や著作権情報。                                                                     |
| セキュリティ/認証     | `ソルト`                   | パスワードのハッシュ化において、ランダムなデータを追加してセキュリティを向上させること。                       |
| セキュリティ/認証     | `ペッパー`                 | パスワードのハッシュ化において、秘密のランダムなデータを追加してセキュリティを向上させること。                 |
| セキュリティ/認証     | `ストレッチング`           | ハッシュ関数の計算を故意に遅らせて、セキュリティを強化すること。                                               |
| セキュリティ/認証     | `JWT`                      | JSON Web Token。JSON形式で情報を安全に伝達するためのトークン。                                                 |
| セキュリティ/認証     | `セキュリティチェーン`     | セキュリティ対策を一本の鎖（チェーン）に見立てた考え方。                                                       |
| 開発/プログラミング   | `糖衣構文`                 | 見た目が美しいが本質的でないプログラミング言語の構文。                                                         |
| 開発/プログラミング   | `Webhook`                  | イベント発生時に外部に通知を送るための仕組み。                                                                 |
| 開発/プログラミング   | `シーケンス`               | 特定の順序で発生する一連のイベントやデータ。                                                                   |
| データ/キャッシュ     | `サロゲートキー`           | 主キーの代わりに使用される仮のキー。                                                                           |
| データ/キャッシュ     | `エビクション`             | キャッシュやデータベースから不要なデータを削除すること。                                                       |
| 信頼性/運用           | `外形監視`                 | システムやネットワークの外部から性能や状態を監視すること。                                                     |
| 信頼性/運用           | `フェイルオーバー`         | システムの障害時に別のシステムに自動的に切り替えること。                                                       |
| 信頼性/運用           | `冪等性`                   | 同じ操作を何度実行しても同じ結果が得られる性質。                                                               |
| テスト/品質           | `ファジングテスト`         | ソフトウェアやシステムにランダムなデータを与えて安全性や信頼性をテストする手法。                               |
| テスト/品質           | `スモークテスト`           | 「とりあえず動くかどうか」「致命的な問題がないか」を超簡単に確認するテストのこと。                             |
| アルゴリズム          | `ダイクストラ法`           | グラフ上のある始点から各頂点への最短経路を求めるアルゴリズムで、重みつき有向グラフや無向グラフに適用される。   |
| ブロックチェーン      | `スマートコントラクト`     | 自動的に実行される契約で、ブロックチェーン技術を使用して条件が満たされたときにコードに基づいて取引を実行する。 |
| AI                    | `ハルシネーション`         | 実際には存在しないものを感じたり認識したりすること。                                                           |
| コミュニケーション    | `LGTM`                     | Looks Good To Me。コードレビューで提案が妥当であるという承認コメント。                                         |
| UX/人間要因           | `ポスチャー`               | ユーザーの姿勢や動作などの態度。                                                                               |
| ドキュメンテーション  | `ポンチ絵`                 | 小さな穴を開けた版によって印刷物を作る技法。                                                                   |

- `エッジケース`：通常はあまり起きないけど、起きるとバグを生みがちな「ギリギリの条件下のケース」のこと。
